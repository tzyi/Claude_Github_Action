1️⃣ main.tsx - 應用進入點

  • createRoot(): 建立 React 應用的根實例
  • <StrictMode>: 開發環境下啟用更嚴格的檢查
  • 掛載到 HTML 中的 #root 元素

2️⃣ types.ts - 型別定義

在 TypeScript 中，定義類型非常重要。這確保資料的結構正確：

export interface WalkingSpot { id: number; name: string; address: string; description: string; rating: number; tags: string[]; image_url: string; } export interface SpotsResponse { city: string; spots: WalkingSpot[]; } export interface CitiesResponse { cities: string[]; }

3️⃣ api.ts - API 呼叫層

與後端通訊的所有邏輯都在這裡：

import axios from "axios"; import type { CitiesResponse, SpotsResponse } from "./types"; const BASE_URL = "http://localhost:5000/api"; export const fetchCities = async (): Promise<CitiesResponse> => { const res = await axios.get<CitiesResponse>(`${BASE_URL}/cities`); return res.data; }; export const fetchSpotsByCity = async (city: string): Promise<SpotsResponse> => { const res = await axios.get<SpotsResponse>( `${BASE_URL}/spots/${encodeURIComponent(city)}` ); return res.data; };

重點: encodeURIComponent() 處理中文等特殊字元，確保 URL 正確編碼。

4️⃣ App.tsx - 主應用組件

這是應用的核心邏輯組件。讓我們一步步分解：

📌 State 管理

const [cities, setCities] = useState<string[]>([]); const [selectedCity, setSelectedCity] = useState<string>(""); const [spots, setSpots] = useState<WalkingSpot[]>([]); const [loading, setLoading] = useState(false); const [error, setError] = useState<string | null>(null);

5 個 State 變數，用來管理應用的狀態：

  • cities: 所有可用的縣市
  • selectedCity: 用戶選擇的縣市
  • spots: 該縣市的散步地點
  • loading: 是否正在載入資料
  • error: 錯誤訊息

🎣 useEffect Hook - 載入縣市

useEffect(() => { fetchCities() .then((data) => setCities(data.cities)) .catch(() => setError("無法載入縣市資料，請確認後端服務是否啟動。")); }, []); // 空的依賴陣列 = 只在組件掛載時執行一次

說明: 應用啟動時，立即呼叫 fetchCities() 取得縣市列表。

🎣 useEffect Hook - 當選擇縣市時

useEffect(() => { if (!selectedCity) return; setLoading(true); setError(null); fetchSpotsByCity(selectedCity) .then((data) => setSpots(data.spots)) .catch(() => setError("無法載入散步地點資料。")) .finally(() => setLoading(false)); }, [selectedCity]); // 當 selectedCity 變更時執行

說明: 當用戶選擇不同的縣市時，自動取得該縣市的散步地點。

🎨 JSX 渲染

return ( <div className="app"> <header className="header">...</header> <main className="main"> <!-- 縣市選擇器 --> <select value={selectedCity} onChange={...}>...</select> <!-- 條件渲染 --> {error && <div className="error-msg">...</div>} {loading && <div className="loading">...</div>} {spots.length > 0 && <SpotCard spot={spot} />} </main> <footer className="footer">...</footer> </div> )

條件渲染: 根據狀態顯示不同的 UI（載入中、錯誤、空狀態、資料）。

5️⃣ SpotCard.tsx - 散步地點卡片組件

const StarRating = ({ rating }: { rating: number }) => { const full = Math.floor(rating); const half = rating - full >= 0.5; return ( <span className="star-rating"> {Array.from({ length: 5 }, (_, i) => { if (i < full) return <span key={i}>★</span>; if (i === full && half) return <span key={i} style={{ opacity: 0.5 }}>★</span>; return <span key={i} style={{ opacity: 0.2 }}>★</span>; })} </span> ); }; export default function SpotCard({ spot }: Props) { return ( <div className="spot-card"> <img src={spot.image_url} alt={spot.name} /> <div className="spot-body"> <h3>{spot.name}</h3> <StarRating rating={spot.rating} /> <p>📍 {spot.address}</p> <p>{spot.description}</p> <div className="spot-tags"> {spot.tags.map((tag) => ( <span key={tag} className="tag">{tag}</span> ))} </div> </div> </div> ); }

特色: 子組件 StarRating 用來顯示星級評分，支援半星評分。

⚛️ React 核心概念速查

1. JSX - JavaScript + XML

允許在 JavaScript 中寫類似 HTML 的語法：

<div className="header"> <h1>{title}</h1> <!-- {} 中是 JavaScript 表達式 --> </div>

2. State (狀態)

useState Hook - 用來存儲組件的狀態
當 State 改變時，組件會自動重新渲染 (Re-render)
State 更新是異步的，不會立即發生

3. Effects (副作用)

useEffect Hook - 用來執行副作用 (如 API 呼叫)
依賴陣列 [] - 控制何時執行 Effect
[] = 只執行一次 (組件掛載時)
[dep1, dep2] = dep1 或 dep2 改變時執行
無依賴陣列 = 每次渲染都執行

4. Props (屬性)

<SpotCard spot={spot} /> <!-- 父組件傳遞 prop --> function SpotCard({ spot }: Props) { // 子組件接收 prop return <div>{spot.name}</div> }

5. 條件渲染

{error && <div className="error-msg">{error}</div>} {loading && <div className="loading">載入中...</div>} {spots.length > 0 && <SpotCard spot={spot} />}

6. 列表渲染

{spots.map((spot) => ( <SpotCard key={spot.id} spot={spot} /> ))}

⚠️ 重要: 列表中每個元素都需要唯一的 key 屬性。

7. 事件處理

<select onChange={(e) => setSelectedCity(e.target.value)}> ... </select>

✨ TypeScript 優勢

為什麼使用 TypeScript？

  • 型別檢查: 捕捉開發時的錯誤，而非運行時
  • 自動完成: IDE 能更好地提示和自動完成
  • 程式碼文件: 型別本身就是文件
  • 重構安全: 改變代碼時更有信心

TypeScript vs JavaScript 對比

❌ JavaScript

function fetchSpots(city) { // 不知道 city 的類型 // 不知道返回值是什麼 return axios.get(url); }

✅ TypeScript

function fetchSpots(city: string): Promise<SpotsResponse> { // 清楚的型別檢查 return axios.get(url); }

  • Frontend 在 Port 5173 運行嗎？
  • api.ts 中的 BASE_URL 正確指向 http://localhost:5000/api？
  • 瀏覽器開發者工具中有 CORS 錯誤嗎？

  • 選擇縣市後能顯示散步地點嗎？

  • 頁面在手機上響應式嗎？

❓ 常見問題

Q: 頁面顯示「CORS 錯誤」

A: 確認後端 (app.py) 中有啟用 CORS:

from flask_cors import CORS CORS(app)

Q: 選擇縣市後沒有顯示資料

A: 檢查：

  • 後端是否運行在 Port 5000？
  • 選擇器的 onChange 事件是否正確？
  • 瀏覽器控制台是否有錯誤？
  • API 呼叫是否成功？（用 Network 標籤檢查）

Q: 樣式沒有應用

A: 檢查：

  • CSS 檔案是否正確導入？
  • class 名稱是否拼寫正確？
  • 瀏覽器快取是否需要清除？ (Ctrl+Shift+R)

Q: 星級評分顯示不正確

A: 檢查 SpotCard.tsx 中 StarRating 組件的邏輯是否計算正確。
