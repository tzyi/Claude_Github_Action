📁 檔案結構說明

main.py - 程式進入點

def main(): print("Hello from backend!") if __name__ == "__main__": main()

這是專案的進入點（雖然實際上不會被執行），只是一個簡單的測試檔案。真正執行的是 app.py。

app.py - Flask 應用主檔案

from flask import Flask, jsonify, request from flask_cors import CORS from data import WALKING_SPOTS, CITIES app = Flask(__name__) CORS(app) # 允許跨域請求

CORS 設定很重要！這允許前端（運行在不同 Port）向後端發送請求。

data.py - 資料檔

包含兩個重要資料結構：

  • WALKING_SPOTS: 字典，以縣市名稱為 Key，包含該縣市的所有散步地點
  • CITIES: 列表，所有可用的縣市名稱

🔌 API 端點詳解

Backend 提供了 3 個 REST API 端點：

端點 方法 說明 回應
/api/cities GET 取得所有可用的縣市列表 { "cities": [...] }
/api/spots GET 取得所有散步地點（所有縣市） { "spots": {...} }
/api/spots/<city> GET 取得特定縣市的散步地點 { "city": "...", "spots": [...] }

API 使用範例

// 1. 取得所有縣市 GET http://localhost:5000/api/cities 回應: { "cities": ["台北市", "新北市", "台中市", ...] } // 2. 取得台北市的散步地點 GET http://localhost:5000/api/spots/台北市 回應: { "city": "台北市", "spots": [ { "id": 1, "name": "大安森林公園", "address": "台北市大安區新生南路二段1號", "description": "台北市最大的都市公園...", "rating": 4.8, "tags": ["草地", "步道", "寵物友善"], "image_url": "https://picsum.photos/..." }, ... ] }

🎯 Backend 的關鍵概念

1. Flask 框架

輕量級的 Python Web 框架，用來建構 REST API。主要特點：

  • 簡單易懂，適合小型項目
  • 使用裝飾器 @app.route() 定義路由
  • 自動將 Python 物件轉換為 JSON
  • 支援 HTTP 方法 (GET, POST, PUT, DELETE 等)

2. CORS (跨域資源共享)

由於前端和後端運行在不同的 Port，需要啟用 CORS：

from flask_cors import CORS CORS(app) # 這一行很重要！

沒有這一行，前端會遇到「CORS 錯誤」，無法訪問 API。

3. 路由 (Routes)

每個 @app.route() 定義一個 API 端點。當用戶訪問某個 URL 時，對應的函數會被執行。

4. 狀態碼 (HTTP Status Code)

  • 200: 成功 (預設)
  • 404: 未找到 (如找不到該縣市)
  • 500: 伺服器錯誤

💡 Backend 最佳實踐

  • 資料與邏輯分離: data.py 只負責存放資料，app.py 負責業務邏輯
  • 錯誤處理: 檢查請求參數是否有效，回傳適當的錯誤訊息和狀態碼
  • URL 編碼: 前端使用 encodeURIComponent(city) 處理特殊字元 (如中文)
  • CORS 設定: 必須啟用，否則前端無法訪問
  • JSON 回應: 使用 jsonify() 自動轉換為 JSON 格式

🚀 如何運行 Backend

步驟 1: 進入 backend 目錄

cd backend

步驟 2: 啟用虛擬環境 (如果未啟用)

# Windows .venv\Scripts\activate # macOS/Linux source .venv/bin/activate

步驟 3: 執行 Flask 應用

python app.py # 或 flask run

預期輸出

* Running on http://127.0.0.1:5000 (Press CTRL+C to quit)
⚠️ 重要: Backend 必須在 Port 5000 運行，前端才能訪問。如果需要改變 Port，也要更新 frontend 的 api.ts 檔案。

✅ 快速檢查清單

  • Backend 在 Port 5000 運行嗎？
  • app.py 中有啟用 CORS 嗎？
  • API 端點是否正確 (/api/cities, /api/spots/<city>)?
  • 資料在 data.py 中正確定義嗎？
  • 前端 api.ts 中的 BASE_URL 指向正確的位址？
  • 使用瀏覽器訪問 http://localhost:5000/api/cities，看是否能取得資料？
